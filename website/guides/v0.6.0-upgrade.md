# Upgrading to edikt v0.6.0

v0.6.0 adds the PRD redesign (split markdown + YAML), the structured evaluator-gates contract, the Go-based tier-2 install, and a pre-push hook that enforces the security invariants from v0.5.0. The upgrade is non-breaking for projects that don't actively use PRDs or specs — but if you do, this is the meaningful release.

## Before you upgrade

Two things to do up front:

1. **Commit or stash in-flight work.** The upgrade rewrites `commands/`, `templates/`, and a few `.edikt/` paths. Nothing in your `docs/` is touched.
2. **Review your hook scripts** if you've added custom hooks. The new pre-push hook enforces INV-003 (no shell JSON concatenation) — see [Breaking changes](#breaking-changes) below.

## Upgrade command

```bash
edikt upgrade --yes
```

Or run interactively for the activation prompt. The launcher and payload update independently — the launcher only needs Homebrew or a fresh install when it changes.

## What changes

### New commands

Three new entry points:

- [`/edikt:sdlc:discovery`](/commands/sdlc/discovery) — structured uncertainty reduction before PRD authoring. Four-section doc (Known, Unknown, Kill Criteria, Discovery Plan).
- [`/edikt:prd:review`](/commands/prd/review) — re-score a PRD against the rubric, report sidecar drift, broken refs, and unstarted FRs.
- [`/edikt:spec:review`](/commands/spec/review) — re-score a SPEC, verify FR coverage from the linked PRD, and check AC pass-through integrity.

The existing `/edikt:sdlc:prd` and `/edikt:sdlc:spec` are rewritten — see the next section.

### PRD v2 (with v1 grandfather)

`/edikt:sdlc:prd` now produces a split artifact: `PRD-NNN-<slug>.md` (narrative) plus `PRD-NNN-<slug>.yaml` (sidecar with FRs, ACs, protections, sync hashes, and revision history).

```text
docs/product/prds/
├── PRD-005-webhook-delivery.md     ← what humans read
└── PRD-005-webhook-delivery.yaml   ← source of truth
```

Five forcing questions are asked at authoring time — problem behind the problem, evidence, north + counter metric, what must NOT change, riskiest assumption. They're not skippable. Rigor calibration (`solo | team | platform`) sets the evaluator threshold.

The full walkthrough is in the [PRD v2 Deep Dive](./prd-v2.md).

**v1 PRDs (no sidecar) continue to work.** SPEC and review commands detect missing sidecars and branch to a reduced flow. There's no forced migration — re-authoring with `/edikt:sdlc:prd PRD-NNN` regenerates the PRD in v2 shape only when you want it.

### SPEC source flexibility

`/edikt:sdlc:spec` now accepts three source types:

```bash
/edikt:sdlc:spec PRD-005           # full FR coverage check
/edikt:sdlc:spec BRAIN-007         # advisory checklist from a brainstorm
/edikt:sdlc:spec "harden the rate limiter"   # free-text, self-contained
```

The sidecar carries exactly one of `source_prd`, `source_brainstorm`, or `source_prompt` — traceability is unambiguous either way.

### Plan model assignment

Plan files now declare a `model:` at plan level and per-phase override:

```yaml
---
type: plan
model: claude-sonnet-4-6
phases:
  - id: 1
    model: claude-haiku-4
  - id: 2
    # inherits plan-level default
---
```

Inheritance: per-phase model → plan-level → `defaults.plan_model` in `.edikt/config.yaml` → `claude-sonnet-4-6`.

Set the project default once:

```bash
/edikt:config set defaults.plan_model claude-sonnet-4-6
```

### Plan-scoped focused context

`/edikt:context --depth=focused` now scans the active plan phase for `PRD-NNN` and `SPEC-NNN` identifiers and loads only those sidecars. Projects with dozens of PRDs no longer pay the context cost of loading them all when the active phase only references two.

### Structured evaluator gates

ADR-023 formalizes the contract between evaluator agents and the `subagent-stop.sh` hook. Verdicts now carry an `evaluator_output` object with explicit `agent`, `severity`, and `findings` fields. The hook reads them directly instead of keyword-matching against prose.

This means gate severity is configured per agent domain in `.edikt/config.yaml`:

```yaml
gates:
  security:    warning
  dba:         critical
  default:     critical
```

The full walkthrough is in [Configuring Evaluator Gates](./evaluator-gates.md).

Legacy unstructured payloads degrade gracefully — they log a warning and fall through to `gates.default`. The legacy path is removed in v0.7.0.

### Pre-push invariant hook

`templates/hooks/pre-push.sh` enforces three invariants on every push:

- **INV-001** — `commands/` and `templates/` contain only `.md`/`.yaml` files
- **INV-002** — accepted ADRs are immutable
- **INV-003** — no shell JSON concatenation in hook scripts

To bypass for a single push (logged):

```bash
EDIKT_BYPASS_PREPUSH=1 git push
```

To disable globally:

```bash
/edikt:config set hooks.pre-push-security false
```

### Doctor checks

Four new checks across PRD/SPEC sidecars: orphaned sidecars, schema version, sidecar drift (`_sync.md_hash` mismatch), and broken refs. Plus fixture characterization rate and recent gate activity. See [/edikt:doctor](/commands/doctor) for the full list.

### `commands/deprecated/` deleted

16 redirect-only command stubs are removed. Intent-based natural-language routing in CLAUDE.md handles discovery — say "write a PRD" and Claude runs `/edikt:sdlc:prd` regardless of which name you remembered.

## Breaking changes

| Change | Action |
|--------|--------|
| `commands/deprecated/` deleted | External scripts sourcing files from there must be updated. Use the namespaced commands directly. |
| Pre-push hook enforces INV-003 | Repos with `echo '{...}'` patterns in hook scripts must switch to `python3 json.dumps`. Run `/edikt:doctor` to identify offenders. |
| `gates:` section in config | New projects get the section automatically. `edikt upgrade` leaves existing configs alone — opt in by setting `gates.security`, `gates.dba`, etc. |
| PRD v2 template | The old `templates/prd.md.tmpl` is renamed to `prd-v1.md.tmpl`. New PRDs use the split-artifact template. Existing v1 PRDs keep loading. |

## "Do nothing" path

If you don't currently use PRDs or specs and don't have custom hooks, the upgrade is a no-op for your workflow:

- No PRDs to migrate
- No hook scripts to audit
- The new commands sit available; you adopt them when you're ready

The rules and agents are unchanged. The governance compile flow is unchanged. Your existing plans keep running.

## Verifying the upgrade

```bash
/edikt:doctor
```

Expected new green lines:

```text
[ok]   PRD/SPEC artifact health — no issues
[ok]   Fixtures fully characterized (or "no fixture artifacts found")
[ok]   Gate activity — no unresolved findings
[ok]   edikt 0.6.0 (installed: 0.6.0)
```

If any check fails, doctor prints the exact remediation command.

## Rolling back

```bash
edikt rollback v0.6.0
```

Restores the previous payload and launcher. Your `docs/` content is untouched — PRDs authored under v2 keep their `.yaml` sidecars (v0.5.x ignores them harmlessly).
