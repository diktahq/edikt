# Upgrading to edikt v0.6.0

v0.6.0's headline change is the **sidecar architecture** ‚Äî governance directives move out of the prose body of every ADR, Invariant Record, and guideline, and into a co-located `<name>.edikt.yaml` sidecar. This is **breaking and mandatory**: every project upgrading from v0.5.x must run a one-time migration before `/edikt:gov:compile` will run again. v0.6.0 also ships a two-phase compile, the new `edikt verify` runner, a narrowed PreToolUse hook scope, the PRD redesign (split markdown + YAML), structured evaluator gates, and a pre-push hook enforcing the v0.5.0 security invariants.

## Before you upgrade

Three things to do up front:

1. **Commit or stash in-flight work.** The upgrade rewrites `commands/`, `templates/`, and a few `.edikt/` paths, and the sidecar migration writes `<artifact>.edikt.yaml` next to every governance `.md` and removes the in-body sentinel block. You want a clean working tree so the diff is reviewable.
2. **Plan time for the migration.** Mechanical lifts (v0.5.x sidecars carrying `source_hash` + `topic:` + `signals:`) are fast. v0.4.3 legacy artifacts and v0.5.x partials (no `topic:` backfilled) require a per-artifact LLM resync ‚Äî budget ~30‚Äì60s per artifact.
3. **Review your hook scripts** if you've added custom hooks. The new pre-push hook enforces INV-003 (no shell JSON concatenation) ‚Äî see [Breaking changes](#breaking-changes) below.

## Upgrade command

```bash
edikt upgrade --yes
```

Or run interactively for the activation prompt. The launcher and payload update independently ‚Äî the launcher only needs Homebrew or a fresh install when it changes.

`/edikt:upgrade` always runs `edikt migrate sidecars --dry-run` first, prints the plan, and prompts for confirmation before applying. Headless invocations (`EDIKT_HEADLESS=1`) defer the migration rather than auto-applying ‚Äî sidecar generation rewrites prose bodies and must never happen silently.

## Governance moves to sidecars (mandatory migration)

Every ADR, Invariant Record, and guideline `.md` now has a co-located `<name>.edikt.yaml` sidecar carrying `topic:`, `signals:`, `directives:`, and `paths:`. The `[edikt:directives:start]‚Ä¶[edikt:directives:end]` block is removed from the prose body.

This is captured in [ADR-027](https://github.com/diktahq/edikt/blob/v0.6.0/docs/architecture/decisions/ADR-027-sidecar-architecture-for-governance-metadata.md) (supersedes ADR-008). The boundary between human-owned bytes (`.md`) and tool-owned bytes (`.edikt.yaml`, `.claude/rules/governance/*.md`) is now structural rather than definitional.

**`/edikt:gov:compile` refuses to run on a pre-migration project.** It exits non-zero with a single-line error directing you to `edikt migrate sidecars`. There is no fallback to in-body sentinel parsing.

### Run the migration

```bash
edikt migrate sidecars --dry-run    # plan; required first
edikt migrate sidecars --apply      # execute the plan
```

`--apply` refuses unless `--dry-run` was run in this directory within the last **24 hours** (the gate file is at `.edikt/state/migration-dry-run.json`). Pass `--force` to bypass; otherwise re-run `--dry-run`. The window exists because sidecar generation is destructive on the prose body ‚Äî you should not apply a plan you haven't reviewed today.

The migration is idempotent and fence-aware. Three lift paths:

| Detected schema | Lift path | Cost |
|---|---|---|
| v0.5.x full (`source_hash` + `topic:` + `signals:`) | Mechanical mapping | No LLM; fast |
| v0.5.x partial (`source_hash:` only, `topic:` never backfilled) | LLM resync per artifact via the locked `sidecar-extractor` agent | ~30‚Äì60s per artifact |
| v0.4.3 legacy (flat `content_hash:` + directive list) | Mechanical extract + LLM `topic`/`signals` derivation | One LLM call per artifact |

See the [Sidecar Migration](./sidecar-migration.md) walkthrough for the complete flow, edge cases, and rollback path.

### INV-005 narrows

[INV-005](https://github.com/diktahq/edikt/blob/v0.6.0/docs/architecture/invariants/INV-005-managed-region-integrity.md) (managed-region integrity) narrows in v0.6.0 to `CLAUDE.md` and `settings.json` only. Governance artifacts (`.md` files in `decisions/`, `invariants/`, `guidelines/`) are **never** managed regions because edikt does not write to them ‚Äî the sidecar owns the structured content. The PreToolUse hook and `Edit`/`Write` overlap guards stop firing on governance files, eliminating an entire false-positive class on documentation pages.

### Two-phase compile

[ADR-028](https://github.com/diktahq/edikt/blob/v0.6.0/docs/architecture/decisions/ADR-028-two-phase-compile-resync-merge.md) amends ADR-020. `/edikt:gov:compile` now runs in two phases:

- **Phase A ‚Äî Resync (conditional, LLM-backed).** Fires only when one or more sidecars are stale (their parent `.md` body hash has drifted). Dispatches the per-artifact `:compile` subagent at concurrency 8, continue-on-error, with mandatory progress UI on stderr. No SLO.
- **Phase B ‚Äî Merge (always, deterministic).** Reads the sidecars, validates against the JSON schema, groups by topic, renders. Pure ‚Äî no LLM, no `Task`/`Agent` dispatch, enforced by a static-analysis test in CI. Preserves ADR-020's latency budget (`<5s` cold, `<500ms` no-op, `<2s` --check).

What you'll notice: the **first** compile after a doc edit pays the Phase A cost; subsequent compiles run sub-second on Phase B alone. `--check` skips Phase A entirely and exits 1 with a stale-sidecar list ‚Äî CI-safe.

The `--legacy` flag retains the v0.5.x in-body parsing path for the v0.5.x ‚Üí v0.6.0 transition window only and is slated for removal in v0.7.0.

### `edikt verify` runner

A new tier-2 binary subcommand walks a plan's criteria sidecar, runs every `verify:` shell command, and writes a JSON+text report under `.edikt/state/verify/`:

```bash
edikt verify <plan-id>              # all phases
edikt verify <plan-id> --phase 4b   # one phase
edikt verify <plan-id> --json       # machine-readable
```

Exit codes: `0` all-pass, `1` failures or timeouts, `2` sidecar missing/malformed, `3` invalid args. `/edikt:sdlc:plan` invokes the runner before flipping a phase row to `done` ‚Äî verification becomes structural, not advisory.

### PreToolUse hook scope narrows

The PreToolUse hook (`templates/hooks/pre-tool-use.sh`) used to scan every `Edit`/`Write` for managed-region overlap. v0.6.0 narrows the in-scope path set to `CLAUDE.md`, `settings.json`, and `.edikt/`. Edits to governance `.md` files no longer trip the hook ‚Äî the sidecar is the source of truth and the prose body is freely editable.

## What changes

### New commands

Three new entry points:

- [`/edikt:sdlc:discovery`](/commands/sdlc/discovery) ‚Äî structured uncertainty reduction before PRD authoring. Four-section doc (Known, Unknown, Kill Criteria, Discovery Plan).
- [`/edikt:prd:review`](/commands/prd/review) ‚Äî re-score a PRD against the rubric, report sidecar drift, broken refs, and unstarted FRs.
- [`/edikt:spec:review`](/commands/spec/review) ‚Äî re-score a SPEC, verify FR coverage from the linked PRD, and check AC pass-through integrity.

The existing `/edikt:sdlc:prd` and `/edikt:sdlc:spec` are rewritten ‚Äî see the next section.

### PRD v2 (with v1 grandfather)

`/edikt:sdlc:prd` now produces a split artifact: `PRD-NNN-<slug>.md` (narrative) plus `PRD-NNN-<slug>.yaml` (sidecar with FRs, ACs, protections, sync hashes, and revision history).

```text
docs/product/prds/
‚îú‚îÄ‚îÄ PRD-005-webhook-delivery.md     ‚Üê what humans read
‚îî‚îÄ‚îÄ PRD-005-webhook-delivery.yaml   ‚Üê source of truth
```

Five forcing questions are asked at authoring time ‚Äî problem behind the problem, evidence, north + counter metric, what must NOT change, riskiest assumption. They're not skippable. Rigor calibration (`solo | team | platform`) sets the evaluator threshold.

The full walkthrough is in the [PRD v2 Deep Dive](./prd-v2.md).

**v1 PRDs (no sidecar) continue to work.** SPEC and review commands detect missing sidecars and branch to a reduced flow. There's no forced migration ‚Äî re-authoring with `/edikt:sdlc:prd PRD-NNN` regenerates the PRD in v2 shape only when you want it.

### SPEC source flexibility

`/edikt:sdlc:spec` now accepts three source types:

```bash
/edikt:sdlc:spec PRD-005           # full FR coverage check
/edikt:sdlc:spec BRAIN-007         # advisory checklist from a brainstorm
/edikt:sdlc:spec "harden the rate limiter"   # free-text, self-contained
```

The sidecar carries exactly one of `source_prd`, `source_brainstorm`, or `source_prompt` ‚Äî traceability is unambiguous either way.

### Plan model assignment

Plan files now declare a `model:` at plan level and per-phase override:

```yaml
---
type: plan
model: claude-sonnet-4-6
phases:
  - id: 1
    model: claude-haiku-4
  - id: 2
    # inherits plan-level default
---
```

Inheritance: per-phase model ‚Üí plan-level ‚Üí `defaults.plan_model` in `.edikt/config.yaml` ‚Üí `claude-sonnet-4-6`.

Set the project default once:

```bash
/edikt:config set defaults.plan_model claude-sonnet-4-6
```

### Plan-scoped focused context

`/edikt:context --depth=focused` now scans the active plan phase for `PRD-NNN` and `SPEC-NNN` identifiers and loads only those sidecars. Projects with dozens of PRDs no longer pay the context cost of loading them all when the active phase only references two.

### Structured evaluator gates

ADR-023 formalizes the contract between evaluator agents and the `subagent-stop.sh` hook. Verdicts now carry an `evaluator_output` object with explicit `agent`, `severity`, and `findings` fields. The hook reads them directly instead of keyword-matching against prose.

This means gate severity is configured per agent domain in `.edikt/config.yaml`:

```yaml
gates:
  security:    warning
  dba:         critical
  default:     critical
```

The full walkthrough is in [Configuring Evaluator Gates](./evaluator-gates.md).

Legacy unstructured payloads degrade gracefully ‚Äî they log a warning and fall through to `gates.default`. The legacy path is removed in v0.7.0.

### Pre-push invariant hook

`templates/hooks/pre-push.sh` enforces three invariants on every push:

- **INV-001** ‚Äî `commands/` and `templates/` contain only `.md`/`.yaml` files
- **INV-002** ‚Äî accepted ADRs are immutable
- **INV-003** ‚Äî no shell JSON concatenation in hook scripts

To bypass for a single push (logged):

```bash
EDIKT_BYPASS_PREPUSH=1 git push
```

To disable globally:

```bash
/edikt:config set hooks.pre-push-security false
```

### Doctor checks

Four new checks across PRD/SPEC sidecars: orphaned sidecars, schema version, sidecar drift (`_sync.md_hash` mismatch), and broken refs. Plus fixture characterization rate and recent gate activity. See [/edikt:doctor](/commands/doctor) for the full list.

### `commands/deprecated/` deleted

16 redirect-only command stubs are removed. Intent-based natural-language routing in CLAUDE.md handles discovery ‚Äî say "write a PRD" and Claude runs `/edikt:sdlc:prd` regardless of which name you remembered.

## Breaking changes

| Change | Action |
|--------|--------|
| Sidecar architecture (ADR-027, supersedes ADR-008) | Run `edikt migrate sidecars --dry-run` then `--apply`. `/edikt:gov:compile` refuses to run until applied. |
| INV-005 narrows to `CLAUDE.md` + `settings.json` | No action ‚Äî the narrowing eliminates false-positive PreToolUse blocks on governance edits. |
| Two-phase compile (ADR-028, amends ADR-020) | No action for steady-state. First compile after a doc edit runs Phase A; subsequent runs are Phase B only. |
| `commands/deprecated/` deleted | External scripts sourcing files from there must be updated. Use the namespaced commands listed below. |
| Pre-push hook enforces INV-003 | Repos with `echo '{...}'` patterns in hook scripts must switch to `python3 json.dumps`. Run `/edikt:doctor` to identify offenders. |
| `gates:` section in config | New projects get the section automatically. `edikt upgrade` leaves existing configs alone ‚Äî opt in by setting `gates.security`, `gates.dba`, etc. |
| PRD v2 template | The old `templates/prd.md.tmpl` is renamed to `prd-v1.md.tmpl`. New PRDs use the split-artifact template. Existing v1 PRDs keep loading. |
| `--legacy` flag on `gov compile` | Transitional only ‚Äî the in-body parsing path is removed in v0.7.0. Migrate sidecars and stop using `--legacy`. |

## "Do nothing" path

If you don't currently use PRDs or specs and don't have custom hooks, the upgrade is a no-op for your workflow:

- No PRDs to migrate
- No hook scripts to audit
- The new commands sit available; you adopt them when you're ready

The rules and agents are unchanged. **The governance compile flow itself is changing** ‚Äî even projects with zero PRDs/specs must run the sidecar migration before `/edikt:gov:compile` will run again. Your existing plans keep running unchanged.

## Verifying the upgrade

```bash
/edikt:doctor
```

Expected new green lines:

```text
[ok]   PRD/SPEC artifact health ‚Äî no issues
[ok]   Fixtures fully characterized (or "no fixture artifacts found")
[ok]   Gate activity ‚Äî no unresolved findings
[ok]   edikt 0.6.0 (installed: 0.6.0)
```

If any check fails, doctor prints the exact remediation command.

## Rolling back

```bash
edikt rollback v0.6.0
```

Restores the previous payload and launcher. Your `docs/` content is untouched ‚Äî PRDs authored under v2 keep their `.yaml` sidecars (v0.5.x ignores them harmlessly).
