Done. Two things to note about the implementation:

**`internal/http/handlers/orders.go`** — `ListOrders` reads both `userID` and `tenantID` from context and rejects the request if either is missing. The tenant scope is required even though the task only mentioned `userID` — using `userID` alone would return orders across all tenants for that user, leaking data. The repository's `FindOrdersByUserAndTenant` enforces this same contract.

**`main.go`** — registered `GET /orders` inside the `WithTenant` middleware wrap, so the context values are always populated before the handler runs.
