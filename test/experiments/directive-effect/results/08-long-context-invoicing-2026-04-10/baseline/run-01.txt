## Invoicing feature — what was built

### New files

**`internal/domain/invoice.go`**
Two types: `InvoiceLineItem` (product ID, quantity, unit price, line subtotal) and `Invoice` (header with order ref, totals, and a slice of line items). Money stays in cents throughout — same convention as the rest of the codebase.

**`internal/repository/invoices.go`**
Three public methods:
- `Create` — wraps the two-table insert in a transaction. The invoice header goes in first (to get the generated UUID), then each line item is inserted referencing it. `tx.Rollback()` is deferred so any mid-flight failure rolls back both tables atomically.
- `GetByID` — fetches the invoice header filtered by `id + tenant_id` (tenant isolation, same pattern as `orders.go`), then a second query loads its line items.
- `ListByUser` — returns invoice headers only, newest-first. Line items are intentionally omitted from the list to avoid an N+1 scan over potentially many invoices.

**`internal/service/invoices.go`**
`InvoiceService` with three methods that mirror the repo layer but add the domain logic: `CreateForOrder` builds the `Invoice` from raw `[]CartItem`, computes each line's subtotal, sums to the invoice subtotal, applies `subtotal * 20 / 100` for tax (integer arithmetic, no float), adds to get the total, then calls the repo.

**`internal/handler/invoices.go`**
- `GET /invoices` → `ListInvoices`: reads `UserIDKey` from context, calls `ListForUser`, serialises. Returns `[]` not `null` when empty.
- `GET /invoices/{id}` → `GetInvoice`: uses Go 1.22's `r.PathValue("id")`, maps `sql.ErrNoRows` → 404, everything else → 500.
- Shared `writeError` helper sets the `Content-Type` header before writing the status code — avoids the subtle bug where setting headers after `WriteHeader` is a no-op.

**`migrations/002_invoices.sql`**
Two tables: `invoices` and `invoice_line_items`. Line items have `ON DELETE CASCADE` so they're cleaned up if an invoice is ever removed. Includes a composite index on `(user_id, tenant_id, created_at DESC)` that exactly matches the list query's WHERE + ORDER BY, and an index on `invoice_id` for the detail lookup.

### Modified files

**`internal/repository/carts.go`**
`GetByUser` previously fetched the cart row but left `Items` nil, so the total calculation in `CheckoutService` always produced zero. Fixed: it now calls the new `GetItems` helper after loading the cart header. `GetItems` is also public so callers can load items independently if needed.

**`internal/service/checkout.go`**
`CheckoutService` gains an `invoices *InvoiceService` field. `Checkout` now returns `(*domain.Invoice, error)` instead of `error`. After the order is created, it calls `invoices.CreateForOrder` with the order ID and the cart items. The structured log at the end includes subtotal, tax, and total so every checkout is fully observable.

**`internal/handler/checkout.go`**
Updated to receive the invoice return value and write it as the JSON response body. Clients now get the full invoice (with line items and tax breakdown) immediately after a successful checkout — no second request needed.

**`cmd/main.go`**
Three new lines for the invoice wiring: `NewInvoicesRepo` → `NewInvoiceService` → `NewInvoicesHandler`. Two new routes registered on the authenticated mux: `GET /invoices` and `GET /invoices/{id}`. `NewCheckoutService` now takes `invoiceSvc` as its third argument.
