    id              TEXT PRIMARY KEY,

);

### Design decisions made

- **Tax is calculated in the service layer**, not the DB or handler ‚Äî the constant lives in `domain` so it's the single source of truth.
- **`Create` uses a transaction** ‚Äî invoice header and all items commit atomically; a partial failure rolls back cleanly.
- **`ListByUser` does not load line items** ‚Äî avoids N+1 queries; the list endpoint is for summaries, `GetByID` is for full detail.
- **`GetByID` scopes by both `tenant_id` and `user_id`** ‚Äî a user in tenant A cannot read another user's invoice even within the same tenant.
- **New repo methods follow the governance rule** ‚Äî `tenantID string` is passed explicitly; context is never inspected inside the repository layer for the new code.
- **All `slog` calls include `"tenant_id", tid`** ‚Äî INV-012 compliant throughout.
