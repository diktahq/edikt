**Handler flow:**
1. Fetch order by ID (tenant-scoped via repo)
2. Verify the authenticated user owns the order — returns `403` otherwise (disguised as 404 to avoid leaking existence)
3. Reject if status is not `paid` → `422`
4. Reject if `createdAt` is more than 1 hour ago → `422`
5. Call `stripe.Refund()` with idempotency key `order-{id}-refund` → `502` on failure
6. Call `MarkCancelled()` which only updates rows still in `paid` status — returns `409` if a concurrent cancellation beat this request
7. Enqueue cancellation email (non-fatal if enqueueing fails, consistent with the existing confirmation email pattern)
8. Return `200` with the updated order
