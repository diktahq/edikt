**`DELETE /orders/{id}`** — requires a valid auth token. Returns:
- `404` — order not found (or belongs to a different tenant)
- `403` — authenticated user doesn't own this order
- `409` — order isn't in `paid` state, or the 1-hour window has passed
- `502` — Stripe refund failed (order left untouched)
- `500` — refund succeeded but DB update failed (logged for reconciliation)
- `204` — success
