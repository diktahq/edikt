## Key design decisions

**`SELECT FOR UPDATE SKIP LOCKED`** (`db.go:53‚Äì62`)
Each poll runs this inside an `UPDATE ‚Ä¶ WHERE id IN (SELECT ‚Ä¶ FOR UPDATE SKIP LOCKED)`. PostgreSQL atomically skips any rows another transaction has locked, so multiple replicas can poll simultaneously without double-claiming jobs.

**Semaphore as concurrency gate** (`scheduler.go:62‚Äì67`)
```go
available := cap(s.sem) - len(s.sem)   // free slots right now
s.sem <- struct{}{}                     // acquire before launch
go s.execute(ctx, job)                  // goroutine releases on exit
```
The poll only claims as many jobs from Postgres as there are free slots ‚Äî no over-claiming, no wasted UPDATE round-trips.

**`db.ErrNotFound` sentinel** (`db.go:14`)
`pgx.ErrNoRows` is mapped to a local sentinel in the `db` package. The HTTP handler checks `errors.Is(err, db.ErrNotFound)` ‚Äî the `api` package never imports `pgx`.

**Go 1.22 `ServeMux`** (`api/handler.go:22‚Äì25`)
`"GET /jobs/{id}"` and `"POST /jobs"` are method-qualified patterns; `r.PathValue("id")` extracts the segment. No external router needed.

## Running it

```bash
# 1. apply the schema
psql $DATABASE_URL -f migrations/001_create_jobs.sql

# 2. start the service
DATABASE_URL="postgres://user:pass@localhost/jobscheduler" go run .

# 3. create a job
curl -s -X POST localhost:8080/jobs \
  -H 'Content-Type: application/json' \
  -d '{"tenant_id":"acme","type":"example","payload":{"key":"value"}}'

# 4. check status
curl -s localhost:8080/jobs/<id>
```

To add a new job type, call `sched.Register("your_type", fn)` in `main.go` before `sched.Run`.
