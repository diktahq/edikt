## Manual verification checklist — run these before tagging v0.5.0

The final two gate steps (full `/edikt:sdlc:audit` + `/edikt:gov:benchmark`) must be run interactively because they invoke Claude agents that aren't reachable from the automation surface. Run them and paste the results into this document under the corresponding section.

### 1. Run the security regression suite

```bash
./test/security/run.sh
```

**Expected:** every file reports `PASS`, final line `✓ all security regression tests passed`.

### 2. Run the existing hook tests (characterization + new fixtures)

```bash
for t in test/unit/hooks/test_*.sh; do bash "$t"; done
```

**Expected:** every test reports `PASS` or the documented `NOTE:` (pre-compact).

### 3. Run `/edikt:sdlc:audit` against the branch

Start a Claude Code session on the `0.5.0-dev` branch and type `/edikt:sdlc:audit`. Paste the resulting report under a new section "## /edikt:sdlc:audit run 2026-04-17" below. Every Critical/High the audit surfaces should either:
- Map to a commit already listed above, OR
- Be a false positive with a written justification, OR
- Be accepted as a v0.5.x follow-up with a tracking note.

### 4. Run `/edikt:gov:benchmark` with the expanded corpus

```bash
/edikt:gov:benchmark
```

Target pass rate:
- ≥98% on Critical/High attack templates (refuse_tool_use, refuse_file_pattern, must_cite*, refuse_edit_matching_frontmatter, evaluator_coercion, sentinel_escape, agent_identity_spoof).
- ≥95% overall.

### 5. Dry-run the release workflow

```bash
gh workflow run release.yml --ref 0.5.0-dev -f dry_run=true
```

Verify the workflow:
- Computes SHA256SUMS correctly.
- Signs with cosign (check for `SHA256SUMS.sig.bundle` in the dry-run log).
- Rejects malformed tags (the tag-shape gate).

### 6. Local cosign verification

Once a test tag is cut (`v0.5.0-rc1` via workflow_dispatch), verify end-to-end:

```bash
TAG=v0.5.0-rc1
curl -fsSL -o /tmp/SHA256SUMS            https://github.com/diktahq/edikt/releases/download/${TAG}/SHA256SUMS
curl -fsSL -o /tmp/SHA256SUMS.sig.bundle https://github.com/diktahq/edikt/releases/download/${TAG}/SHA256SUMS.sig.bundle
cosign verify-blob \
  --bundle /tmp/SHA256SUMS.sig.bundle \
  --certificate-identity-regexp '^https://github\.com/diktahq/edikt/\.github/workflows/release\.yml@refs/tags/v[0-9]+\.[0-9]+\.[0-9]+(-[A-Za-z0-9.-]+)?$' \
  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com' \
  /tmp/SHA256SUMS
```

**Expected:** `Verified OK` on stderr.

## Deferred to v0.5.x

1. **HI-8 final mile** — add `environment: production` with a required reviewer on `merge-tap-pr` in `release.yml`.
2. **ADR-016 launcher signing** — currently SHA256SUMS covers the payload tarball signed, but the raw `bin/edikt` file via `raw.githubusercontent.com/<tag>/bin/edikt` is not yet signed as its own asset. When we build `bin/edikt` as a standalone release asset, add it to SHA256SUMS so `install.sh`'s launcher verify is fully end-to-end.
3. **Grandfather routine runtime** — ADR-018 spec is complete; `bin/edikt upgrade` needs to emit verdict stubs for existing `done` phases. Tracked for v0.5.1.
4. **bin/edikt rollback v0.5.0** — full migration rollback with file restore (current rollback is payload-only).
5. **LOW-3 hash anchor seeding** — compile should emit `[edikt:NAME:sha256]: # <hex>` inside every managed region. INV-005's bootstrap rule (absent → seed) is documented; next compile implementation pass realises it.
6. **MED-4** — move 10+ markdown-embedded Python scripts from `test/integration/test_*.py` into `test/_lib/`.
7. **MED-7, MED-11, LOW-1/2/4/5/6/13** — small one-offs tracked in the plan.
